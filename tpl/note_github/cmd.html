6.查看提交历史

7.查看三个区的文件内容差异对比

(1)git diff

(3)git diff --master
工作区 ----- 版本库

8.撤销操作

git reset HEAD file.name ---- 将缓存区的文件撤销

git checkout file.name ---- 将工作区的文件撤销 先尝试从缓存区还原，再尝试从版本库还原

git commit -m "备注" --amend
----假如进行一次 commit 提交 ，漏提交其他文件 ，可将上次提交撤销 ，将暂存区的文件与上次提交合并到一起进行提交

9.删除操作

git rm file.name
---- 工作区中文件如果已经删掉，暂存区中还存在，可通过该命令行删除暂存区中的该文件
---- 如果工作区中的文件没有删除 ， 该命令执行error

git rm -f file.name
---- 同时删掉工作区与缓存区的文件

git rm -cached file.name
---- 只会把缓存区的文件删掉

10.恢复

git checkout commit_id file.name
----指定文件恢复，恢复工作区中误删的文件
git checkout master
----回到master分支

git reset --hard commit_id
----指定版本恢复，将工作区整个项目还原到该版本
>>HEAD^ ----回到上一版本(HEAD其实就是一个指针)
>>HEAD~<num> ----指定恢复到前 num 个版本

git reflog
----显示版本操作记录

11.同步到远程仓库

git remote
---- 查看仓库名
-v
----查看仓库地址
git push origin(仓库名) master(分支名) ----同步到远程仓
----如果发生冲突则error无法提交 , 解决冲突后再提交(coding前先进行代码的更新)

多人协作解决冲突
git fetch
----从远程仓拉取代码但是并不合并(首选)
git diff master origin/master
----拉取代码后打印区别
git merge origin/master
----手动合并代码
----冲突解决，解决后再commit代码就可以了
git pull
----拉取代码并且自动合并

13.Git 分支

git branch
(1)branch.name
---- 新建一个名称为branch.name的分支
(2)checkout branch.name
---- 切换到 branch.name 的这个分支上
(3)git checkout branch.name
---- 新建一个名称为branch.name的分支 ， 并且直接切换到该分支上
(4)merge branch.name
---- 在master主分支上进行该命令行操作 ， 可将branch.name分支直接合并到主分支上
(5)--merged
---- 查看已合并的分支
(6)--no-merged
---- 查看未合并的分支
(7)-d branch.name
---- 直接删除已经与主分支merged的分支 (8)-D branch.name
---- 强制删除未与主分支merged的分支
(9)merge产生冲突
在master branch解决冲突后 commit 则可将master 和产生冲突的branch 进行合并 ， --merged 可看到两个分支已经合并 ， 但是产生冲突的branch并没有改变
